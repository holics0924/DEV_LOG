# 블로그 글 작성 규칙

## 작성 전 체크
- SVG / 인터랙티브 필요 여부 먼저 사용자에게 물어볼 것

## 헤딩 구조
- `#` — 글 제목 (h1)
- `##` — 대섹션 (h2)
- `###` — 소섹션 (h3, CSS 파란 선 적용됨)
- 내용이 나뉘는 곳엔 반드시 `###` 사용

## 글쓰기 스타일
- `** **` 볼드 강조 사용 금지 — 문장 구조로 강조
- AI 특유의 구조, 이모티콘 없이 자연스러운 기술 글쓰기
- 각 개념마다 "왜 필요한가" 실제 시나리오 포함 — 추상적 설명만 있으면 필요성을 못 느낌
- SVG/그림과 텍스트 절대 겹치지 않게 할 것
- ASCII 다이어그램 적극 활용
- 단락 짧게 쪼개서 호흡 줄이기 — 텍스트만 이어지면 책처럼 읽힘
- 제목은 건조하고 직관적으로, 수사적 표현 불필요
- 깊이는 학부 수업보다 한 단계 더 (PCB 내부 구조, ASID 같은 수준)
- 섹션 간 공백은 `<br><br>` 2개 사용 (3개는 너무 넓음)

## 마지막 문장
- 다음 챕터 개념을 자연스럽게 한 문장으로 던지고 마무리
- "다음 챕터에서 다룬다", "다음 주제다" 같은 직접 명시 금지
- 예시: "OS가 프로세스에게 실행 흐름을 넘기고 다시 빼앗는 메커니즘, 인터럽트가 그 기반에 있다."

## 푸시
- 사용자가 "푸시해"라고 명시할 때만 push
- 작업 완료 후 자동 push 절대 금지
- notes/, curriculum/, quiz/, progress.md 등 학습 내부 파일은 push 금지
- 새 챕터 push 시 index.md 링크 추가 + chapter count 업데이트 반드시 같이 커밋
