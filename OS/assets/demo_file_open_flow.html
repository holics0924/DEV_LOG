파일 열기 흐름 시뮬레이터

open("/home/alice/hello.txt") 호출 시 OS가 거치는 단계

/home/alice/hello.txt
0 / 6
STEP 1
루트(/) 디렉토리 읽기 → "home" 탐색
경로를 왼쪽부터 파싱한다. 루트(/) 디렉토리를 읽어 "home"에 해당하는 inode 번호를 찾는다.
이름inode 번호
home/5
usr/6
etc/7
→ "home" 발견. inode #5 획득.
STEP 2
inode #5 읽기 → /home 디렉토리에서 "alice" 탐색
inode #5를 읽어 /home의 데이터 블록 위치를 얻고, 그 안에서 "alice"를 찾는다.
이름inode 번호
bob/17
alice/18
guest/19
→ "alice" 발견. inode #18 획득.
STEP 3
inode #18 읽기 → /home/alice 디렉토리에서 "hello.txt" 탐색
inode #18을 읽어 /home/alice의 데이터 블록을 얻고, 그 안에서 "hello.txt"를 찾는다.
이름inode 번호
hello.txt42
image.png77
notes/13
→ "hello.txt" 발견. inode #42 획득.
STEP 4
inode #42 읽기 → 메타데이터 + 블록 위치 획득
inode #42에서 파일 크기, 권한, 실제 데이터 블록 번호를 읽는다.
크기3KB
권한rw-r--r--
소유자alice
데이터 블록[7, 23, 101]
→ 블록 #7, #23, #101에 실제 내용이 있음.
STEP 5
데이터 블록 #7, #23, #101 읽기
세 블록을 순서대로 읽어 파일 내용을 조합한다. 블록들은 디스크 어디에나 흩어져 있을 수 있다.
#7
"Hello, "
#23
"Alice!"
#101
"..."
#8
(다른파일)
#44
(비어있음)
완료
파일 내용 반환 + 파일 디스크립터 발급
세 블록의 내용을 조합해 프로세스에 반환한다. OS는 파일 디스크립터(fd)를 발급해 이후 read()/write() 호출에 사용한다.
fd = 3 → "Hello, Alice!..."
총 디스크 접근: 최소 8번 (디렉토리 조회 3 + inode 읽기 3 + 데이터 블록 3)
