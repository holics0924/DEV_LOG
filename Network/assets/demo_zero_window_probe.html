Zero Window Probe (ZWP)

수신 버퍼가 꽉 찼을 때 — 데드락 방지 메커니즘

클라이언트 수신 버퍼 — rwnd
rwnd = 3500 byte
📨 서버가 데이터를 계속 전송 (rwnd 소비 중)
⛔ 수신 버퍼 꽉 참 — 클라이언트 rwnd=0 통보
⏸️ 서버 전송 중단 — 그런데 그냥 기다리면?
🔍 서버가 1-byte Probe 전송 (ZWP)
💬 클라이언트: ACK + rwnd=2000 응답
▶️ 서버 전송 재개 (rwnd만큼)
설명
서버가 데이터를 전송하면서 클라이언트 수신 버퍼를 채우고 있다. ACK에 포함된 rwnd 값이 줄어들고 있다.
